    Ssh.zh

    • {'build': True, 'locale': 'en', 'name': 'English'}

    ssh 入站是一个 SSH 服务器，通过 SSH 隧道支持类似 SOCKS 的代理功能。

    支持功能： - 动态端口转发 (ssh -D) - 本地端口转发 (ssh -L) - 跳板机 / ProxyJump (ssh -J) - 会话转发到目标 SSH 服务器

    结构

    {
  "type": "ssh",
  "tag": "ssh-in",

  ... // 监听字段

  "users": [
    {
      "name": "admin",
      "password": "admin",
      "public_key": [],
      "public_key_path": "",
      "session_address": "",
      "session_port": 22,
      "session_user": "",
      "session_password": "",
      "session_private_key": [],
      "session_private_key_path": "",
      "session_private_key_passphrase": ""
    }
  ],
  "password": "",
  "public_key": [],
  "public_key_path": "",
  "host_key": [],
  "host_key_path": "/etc/sing-box/ssh_host_key",
  "host_key_passphrase": "",
  "server_version": "",
  "session_address": "",
  "session_port": 22,
  "session_user": "",
  "session_password": "",
  "session_private_key": [],
  "session_private_key_path": "",
  "session_private_key_passphrase": ""
}

    监听字段

    参阅 监听字段。

    客户端认证

    SSH 入站支持两种认证方式：密码认证和公钥认证。

    认证优先级

    1. 用户级凭据：如果用户在 users 列表中，优先检查用户的密码/公钥
    2. 全局凭据：回退到全局 password/public_key
    3. 继承：users 列表中没有密码的用户继承全局 password

    认证矩阵

    用户在 users 中 有用户密码 有全局密码 密码认证结果
    是 是 - 使用用户密码
    是 否 是 使用全局密码
    是 否 否 密码认证禁用
    否 - 是 使用全局密码
    否 - 否 "user not found"
    用户在 users 中 有用户公钥 有全局公钥 公钥认证结果
    是 是 - 先检查用户公钥，再检查全局
    是 否 是 检查全局公钥
    是 否 否 "public key rejected"
    否 - 是 检查全局公钥
    否 - 否 "public key rejected"

    字段

    users

    用于认证的 SSH 用户列表。如果配置了全局 password 或 public_key，此字段可选。

    users.name

    用户名。

    users.password

    密码认证的密码。

    如果为空且配置了全局 password，将使用全局密码。

    users.public_key

    用户的公钥列表，用于公钥认证。

    users.public_key_path

    用户 authorized_keys 文件路径。

    users.session_address

    用户会话转发的目标 SSH 服务器地址。

    设置后，该用户的交互式会话将被转发到此 SSH 服务器，而不是全局 session_address。

    users.session_port

    用户目标 SSH 服务器端口。默认为 22。

    users.session_user

    用户目标 SSH 服务器认证的用户名。

    如果为空，将使用客户端连接的用户名（透传）。

    users.session_password

    用户目标 SSH 服务器认证的密码。

    users.session_private_key

    用户目标 SSH 服务器认证的私钥（PEM 格式）。

    users.session_private_key_path

    用户目标 SSH 服务器认证的私钥文件路径。

    users.session_private_key_passphrase

    用户加密私钥的密码。

    password

    全局认证密码。

    • users 列表中没有密码的用户将继承此全局密码
    • 不在 users 列表中的用户可以使用任意用户名配合此密码认证

    public_key

    全局授权公钥。

    任何用户（包括不在 users 列表中的用户）都可以使用这些公钥进行认证。

    public_key_path

    全局 authorized_keys 文件路径。

    host_key

    SSH 主机密钥（PEM 格式）。

    host_key_path

    如果 host_key 为空则必填

    SSH 主机密钥文件路径。如果文件不存在，将自动生成新的 ED25519 密钥。

    host_key_passphrase

    加密主机密钥的密码。

    server_version

    SSH 服务器版本字符串。如果为空，将使用随机版本。

    session_address

    会话转发的目标 SSH 服务器地址。

    设置后，交互式会话（ssh user@host）将被转发到此 SSH 服务器，而不是被拒绝。

    端口转发（-D、-L、-J）不受影响，仍通过路由器路由。

    session_port

    目标 SSH 服务器端口。默认为 22。

    session_user

    目标 SSH 服务器认证的用户名。

    如果为空，将使用客户端连接的用户名（透传）。

    session_password

    目标 SSH 服务器认证的密码。

    当 Agent 转发和私钥认证失败时作为回退。

    session_private_key

    目标 SSH 服务器认证的私钥（PEM 格式）。

    当 Agent 转发失败时作为回退。

    session_private_key_path

    目标 SSH 服务器认证的私钥文件路径。

    session_private_key_passphrase

    加密私钥的密码。

    会话转发

    当客户端请求交互式会话时，sing-box 将其转发到配置的目标 SSH 服务器。

    会话目标优先级

    1. 用户级会话：如果用户在 users 列表中且有 session_address，使用用户的会话配置
    2. 全局会话：否则使用全局 session_address
    3. 拒绝：如果没有配置会话，则拒绝会话请求

    会话认证优先级

    连接目标 SSH 服务器时，按顺序尝试认证方法：

    1. SSH Agent 转发：如果客户端使用了 -A 参数，优先尝试客户端的 agent 密钥
    2. 配置的私钥：尝试 session_private_key（如果已配置）
    3. 配置的密码：尝试 session_password（如果已配置）

    这允许： - 使用 -A 参数的用户使用自己的密钥（保留身份） - 没有 agent 的用户回退到配置的凭据

    会话用户名透传

    如果未配置 session_user： - 客户端的用户名将透传到目标服务器 - 示例：ssh git@proxy → 以 git 用户连接目标

    示例

    基本代理服务器

    {
  "type": "ssh",
  "tag": "ssh-in",
  "listen": ":2222",
  "host_key_path": "/etc/sing-box/ssh_host_key",
  "users": [
    {
      "name": "admin",
      "password": "admin123"
    }
  ]
}

    使用方法： 

    # 通过动态端口转发实现 SOCKS 代理
ssh -D 1080 -N admin@server -p 2222

# 跳板机
ssh -J admin@server:2222 target-host

# 本地端口转发
ssh -L 8080:target:80 -N admin@server -p 2222

    公钥认证

    {
  "type": "ssh",
  "tag": "ssh-in",
  "listen": ":2222",
  "host_key_path": "/etc/sing-box/ssh_host_key",
  "public_key": [
    "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAA..."
  ]
}

    全局密码与用户继承

    {
  "type": "ssh",
  "tag": "ssh-in",
  "listen": ":2222",
  "host_key_path": "/etc/sing-box/ssh_host_key",
  "password": "global_password",
  "public_key_path": "~/.ssh/authorized_keys",
  "users": [
    { "name": "alice" },
    { "name": "bob" },
    { "name": "charlie", "password": "special_password" }
  ]
}

    使用此配置： - alice 和 bob：使用全局密码 global_password 或全局公钥 - charlie：使用自己的密码 special_password，也可以使用全局公钥 - 其他任意用户名：可以使用全局密码或公钥认证

    会话转发

    {
  "type": "ssh",
  "tag": "ssh-in",
  "listen": ":2222",
  "host_key_path": "/etc/sing-box/ssh_host_key",
  "public_key_path": "~/.ssh/authorized_keys",
  "session_address": "192.168.1.100",
  "session_port": 22,
  "session_private_key_path": "~/.ssh/id_ed25519"
}

    使用此配置： - ssh -D 1080 user@server -p 2222 - 动态转发通过路由器 - ssh user@server -p 2222 - 交互式会话转发到 192.168.1.100:22

    用户级会话目标

    {
  "type": "ssh",
  "tag": "ssh-in",
  "listen": ":2222",
  "host_key_path": "/etc/sing-box/ssh_host_key",
  "public_key_path": "~/.ssh/authorized_keys",
  "session_address": "default-server.local",
  "session_private_key_path": "~/.ssh/id_ed25519",
  "users": [
    {
      "name": "alice",
      "session_address": "alice-server.local",
      "session_user": "alice_admin",
      "session_private_key_path": "~/.ssh/alice_key"
    },
    {
      "name": "bob",
      "session_address": "bob-server.local",
      "session_password": "bob_password"
    },
    { "name": "charlie" }
  ]
}

    使用此配置： - alice：会话转发到 alice-server.local，使用 alice_admin 用户和私钥 - bob：会话转发到 bob-server.local，使用 bob 用户和密码 - charlie：会话转发到全局 default-server.local

    SSH Agent 转发

    使用本地 SSH 密钥认证目标服务器：

    {
  "type": "ssh",
  "tag": "ssh-in",
  "listen": ":2222",
  "host_key_path": "/etc/sing-box/ssh_host_key",
  "public_key_path": "~/.ssh/authorized_keys",
  "session_address": "target-server.local",
  "session_password": "fallback_password"
}

    使用方法： 

    # 使用 -A 参数启用 agent 转发
ssh -A user@singbox -p 2222

# 你的本地 SSH agent 将用于认证 target-server.local
# 如果 agent 认证失败，回退到 session_password

    使用此配置： - 使用 -A 参数的客户端：通过 agent 转发使用本地私钥 - 不使用 -A 参数的客户端：回退到 session_password - 私钥永远不会离开你的本地机器（只转发签名）

    Git 代理与动态路由

    使用 sing-box 作为 Git SSH 代理，支持切换后端：

    {
  "type": "ssh",
  "tag": "ssh-in",
  "listen": ":2222",
  "host_key_path": "/etc/sing-box/ssh_host_key",
  "public_key_path": "~/.ssh/authorized_keys",
  "session_address": "127.0.0.1"
}

    路由规则： 

    {
  "rules": [
    {
      "ip_cidr": ["127.0.0.1/32"],
      "outbound": "git-selector"
    }
  ]
}

    使用方法： 

    # 使用 git 用户连接，启用 agent 转发
ssh -A git@singbox -p 2222

# 或在 ~/.ssh/config 中配置
Host singbox-git
    HostName singbox
    Port 2222
    User git
    ForwardAgent yes

# 然后用于 git
git clone singbox-git:user/repo.git

    使用此配置： - session_address 设为 127.0.0.1，通过 git-selector 路由 - 通过面板切换不同的 Git 服务器（GitHub、GitLab 等） - 用户名 git 透传到目标服务器 - 每个用户通过 agent 转发使用自己的 SSH 密钥认证 - Git 提交归属于各自的账户

    混合认证与回退

    {
  "type": "ssh",
  "tag": "ssh-in",
  "listen": ":2222",
  "host_key_path": "/etc/sing-box/ssh_host_key",
  "password": "guest_password",
  "public_key_path": "~/.ssh/authorized_keys",
  "session_address": "internal-server.local",
  "session_password": "internal_password",
  "users": [
    {
      "name": "admin",
      "password": "admin_password",
      "session_address": "admin-server.local",
      "session_private_key_path": "~/.ssh/admin_key"
    }
  ]
}

    使用此配置：

    客户端 sing-box 认证 会话目标 会话认证
    ssh -A admin@... admin_password 或公钥 admin-server.local Agent → admin_key
    ssh admin@... admin_password 或公钥 admin-server.local 仅 admin_key
    ssh -A anyone@... guest_password 或公钥 internal-server.local Agent → internal_password
    ssh anyone@... guest_password 或公钥 internal-server.local 仅 internal_password
